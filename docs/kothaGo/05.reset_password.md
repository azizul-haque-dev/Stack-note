# Part 5 — Reset Password
## POST /api/v1/auth/forgot-password · POST /api/v1/auth/reset-password

---

## Flow

```
── Step 1: Request reset ──────────────────────────────────────────────────────

User → POST /auth/forgot-password { email }
    ↓
Rate limiter (3 req/min per IP — prevent email bombing)
    ↓
Find user by email
    ↓ (always return 200 — don't reveal if email exists)
Generate crypto-random token (32 bytes → hex = 64 chars)
    ↓
Hash token with bcrypt → store in DB with 15-minute expiry
    ↓
Send email with link: https://your-app.com/reset-password?token=<raw_token>
    ↓
Return 200 { message: "If that email is registered, a reset link has been sent." }

── Step 2: Submit new password ────────────────────────────────────────────────

User clicks link → lands on /reset-password?token=xxx
User submits new password

Client → POST /auth/reset-password { token, newPassword }
    ↓
Rate limiter (5 req/min per IP)
    ↓
Zod validation
    ↓
Find the most recent unused, non-expired PasswordResetToken for this user
    ↓
bcrypt.compare(rawToken, storedHash) → invalid? → 400
    ↓
Mark token as used (isUsed = true)
    ↓
bcrypt hash new password (cost 12)
    ↓
Update user.passwordHash
    ↓
Revoke ALL refresh tokens for this user (force re-login on all devices)
    ↓
Return 200 { message: "Password reset successful. Please log in." }
```

## 1. Schema Addition

The `PasswordResetToken` model is in `schema.prisma` (see Part 1). Here's a reminder:

```prisma
model PasswordResetToken {
  id        String   @id @default(auto()) @map("_id") @db.ObjectId
  tokenHash String                        // Raw token NEVER stored
  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  userId    String   @db.ObjectId
  expiresAt DateTime                      // 15 minutes from creation
  isUsed    Boolean  @default(false)      // One-time use
  createdAt DateTime @default(now())

  @@index([userId])
  @@map("password_reset_tokens")
}
```

## 3. Password Reset Service

```typescript
// backend/src/auth/password-reset.service.ts
import crypto from 'crypto';
import * as bcrypt from 'bcrypt';
import { prisma } from '../lib/prisma';
import { usersService } from '../users/users.service';
import { tokensService } from '../tokens/tokens.service';
import { sendPasswordResetEmail } from '../lib/email';

export const passwordResetService = {
  // ─── Step 1: Request a password reset ────────────────────────────────────────
  async requestReset(email: string): Promise<void> {
    const user = await usersService.findByEmail(email);

    // ALWAYS return success — never confirm whether an email is registered.
    // This prevents email enumeration. Process stops here if no user found,
    // but the caller gets the same 200 response either way.
    if (!user) return;

    // Google-only accounts have no password — nothing to reset.
    // Still return silently — don't reveal which provider the account uses.
    if (!user.passwordHash) return;

    if (!user.isActive) return;

    // ── Invalidate any previous unused reset tokens for this user ─────────────
    // Prevents multiple valid reset tokens existing simultaneously.
    await prisma.passwordResetToken.updateMany({
      where: { userId: user.id, isUsed: false },
      data: { isUsed: true },
    });

    // ── Generate a cryptographically secure random token ──────────────────────
    // crypto.randomBytes(32) gives 256 bits of entropy — unguessable.
    // Never use Math.random() or uuid for security tokens.
    const rawToken = crypto.randomBytes(32).toString('hex'); // 64-char hex string

    // Hash before storing — same reason as refresh tokens.
    // Raw token only ever exists in the email link and in memory here.
    const tokenHash = await bcrypt.hash(rawToken, 10);

    // 15 minutes — short enough to limit exposure, long enough for the user to act
    const expiresAt = new Date(Date.now() + 15 * 60 * 1000);

    await prisma.passwordResetToken.create({
      data: {
        tokenHash,
        userId: user.id,
        expiresAt,
      },
    });

    // Send email — fire-and-forget in terms of user response,
    // but await here so errors bubble up to the error handler
    await sendPasswordResetEmail(user.email, user.firstName ?? 'there', rawToken);
  },

  // ─── Step 2: Validate token and reset password ───────────────────────────────
  async resetPassword(rawToken: string, newPassword: string): Promise<void> {
    // ── Find candidates ───────────────────────────────────────────────────────
    // We don't store the raw token, so we can't look up by token directly.
    // Instead: get all recent unused, unexpired tokens and bcrypt.compare each.
    // In practice there should be at most 1 (we invalidated old ones in step 1).
    const candidates = await prisma.passwordResetToken.findMany({
      where: {
        isUsed: false,
        expiresAt: { gt: new Date() },
      },
      include: { user: true },
      orderBy: { createdAt: 'desc' },
      take: 10, // Safety limit — should only ever be 1-2 in normal operation
    });

    // ── Find the matching token ───────────────────────────────────────────────
    let matchedRecord: (typeof candidates)[number] | null = null;

    for (const candidate of candidates) {
      const isMatch = await bcrypt.compare(rawToken, candidate.tokenHash);
      if (isMatch) {
        matchedRecord = candidate;
        break;
      }
    }

    if (!matchedRecord) {
      const err: any = new Error('Invalid or expired reset token');
      err.status = 400;
      throw err;
    }

    // Double-check expiry (paranoia — already filtered in query, but be explicit)
    if (new Date() > matchedRecord.expiresAt) {
      const err: any = new Error('Reset token has expired. Please request a new one.');
      err.status = 400;
      throw err;
    }

    const user = matchedRecord.user;

    if (!user.isActive) {
      const err: any = new Error('Account is deactivated');
      err.status = 401;
      throw err;
    }

    // ── Mark token as used ────────────────────────────────────────────────────
    // Do this BEFORE updating the password — if password update fails,
    // the token is still consumed. User must request a new one.
    // This prevents replay attacks in case of partial failures.
    await prisma.passwordResetToken.update({
      where: { id: matchedRecord.id },
      data: { isUsed: true },
    });

    // ── Hash and save new password ────────────────────────────────────────────
    const saltRounds = parseInt(process.env.BCRYPT_SALT_ROUNDS ?? '12');
    const newHash = await bcrypt.hash(newPassword, saltRounds);

    await usersService.updatePassword(user.id, newHash);

    // ── Revoke ALL refresh tokens ─────────────────────────────────────────────
    // Password changed = all existing sessions are potentially compromised.
    // Force re-login on every device. This is the correct security behaviour.
    await tokensService.revokeUserTokens(user.id);
  },
};
```

## 6. Auth Router — /forgot-password & /reset-password

```typescript
// backend/src/auth/auth.router.ts  (password reset portion)
import { passwordResetService } from './password-reset.service';
import {
  validate,
  forgotPasswordSchema,
  resetPasswordSchema,
} from './auth.validators';
import {
  passwordResetLimiter,
  resetSubmitLimiter,
} from '../middleware/rate-limit';

// ─── POST /auth/forgot-password ──────────────────────────────────────────────
router.post(
  '/forgot-password',
  passwordResetLimiter,
  validate(forgotPasswordSchema),
  async (req: Request, res: Response, next: NextFunction) => {
    try {
      await passwordResetService.requestReset(req.body.email);

      // Always return 200 with a generic message.
      // Never reveal whether the email was found or not.
      return res.json({
        message: 'If that email is registered, a reset link has been sent.',
      });
    } catch (err) {
      next(err);
    }
  },
);

// ─── POST /auth/reset-password ───────────────────────────────────────────────
router.post(
  '/reset-password',
  resetSubmitLimiter,
  validate(resetPasswordSchema),
  async (req: Request, res: Response, next: NextFunction) => {
    try {
      const { token, newPassword } = req.body;
      await passwordResetService.resetPassword(token, newPassword);

      return res.json({
        message: 'Password reset successful. Please log in with your new password.',
      });
    } catch (err) {
      next(err);
    }
  },
);
```

## 7. Full auth.router.ts (Combined)

```typescript
// backend/src/auth/auth.router.ts
import { Router, Request, Response, NextFunction } from 'express';
import { authService } from './auth.service';
import { tokensService } from '../tokens/tokens.service';
import { usersService } from '../users/users.service';
import { passwordResetService } from './password-reset.service';
import {
  validate,
  registerSchema,
  loginSchema,
  googleAuthSchema,
  forgotPasswordSchema,
  resetPasswordSchema,
} from './auth.validators';
import { authenticate, authenticateRefresh, AuthRequest } from '../middleware/authenticate';
import {
  registerLimiter,
  loginLimiter,
  googleLimiter,
  passwordResetLimiter,
  resetSubmitLimiter,
} from '../middleware/rate-limit';

const router = Router();

const IS_PRODUCTION = process.env.NODE_ENV === 'production';

const REFRESH_COOKIE_OPTIONS = {
  httpOnly: true,
  secure: IS_PRODUCTION,
  sameSite: 'strict' as const,
  maxAge: 7 * 24 * 60 * 60 * 1000,
  path: '/api/v1/auth',
};

function isMobile(req: Request) {
  return /ReactNative|Expo|okhttp/.test(req.headers['user-agent'] ?? '');
}

function getMeta(req: Request) {
  return {
    userAgent: req.headers['user-agent'],
    ipAddress: (req.headers['x-forwarded-for'] as string) ?? req.ip,
  };
}

router.post('/register', registerLimiter, validate(registerSchema), async (req, res, next) => {
  try {
    const { accessToken, refreshToken, user } = await authService.register(req.body, getMeta(req));
    if (!isMobile(req)) {
      res.cookie('refresh_token', refreshToken, REFRESH_COOKIE_OPTIONS);
      return res.status(201).json({ accessToken, user });
    }
    return res.status(201).json({ accessToken, refreshToken, user });
  } catch (err) { next(err); }
});

router.post('/login', loginLimiter, validate(loginSchema), async (req, res, next) => {
  try {
    const { accessToken, refreshToken, user } = await authService.login(req.body, getMeta(req));
    if (!isMobile(req)) {
      res.cookie('refresh_token', refreshToken, REFRESH_COOKIE_OPTIONS);
      return res.json({ accessToken, user });
    }
    return res.json({ accessToken, refreshToken, user });
  } catch (err) { next(err); }
});

router.post('/google', googleLimiter, validate(googleAuthSchema), async (req, res, next) => {
  try {
    const { accessToken, refreshToken, user } = await authService.googleAuth(req.body.idToken, getMeta(req));
    if (!isMobile(req)) {
      res.cookie('refresh_token', refreshToken, REFRESH_COOKIE_OPTIONS);
      return res.json({ accessToken, user });
    }
    return res.json({ accessToken, refreshToken, user });
  } catch (err) { next(err); }
});

router.post('/refresh', authenticateRefresh, async (req: AuthRequest, res, next) => {
  try {
    const { accessToken, refreshToken } = await tokensService.rotateRefreshToken(req.user as any, getMeta(req));
    if (!isMobile(req)) {
      res.cookie('refresh_token', refreshToken, REFRESH_COOKIE_OPTIONS);
      return res.json({ accessToken });
    }
    return res.json({ accessToken, refreshToken });
  } catch (err) { next(err); }
});

router.post('/logout', authenticate, async (req: AuthRequest, res, next) => {
  try {
    await tokensService.revokeUserTokens(req.user!.sub, req.user?.tokenId);
    res.clearCookie('refresh_token', { path: '/api/v1/auth' });
    return res.json({ message: 'Logged out successfully' });
  } catch (err) { next(err); }
});

router.get('/me', authenticate, async (req: AuthRequest, res, next) => {
  try {
    const user = await usersService.findById(req.user!.sub);
    if (!user) return res.status(404).json({ message: 'User not found' });
    const { passwordHash, ...safe } = user as any;
    return res.json({ user: safe });
  } catch (err) { next(err); }
});

router.post('/forgot-password', passwordResetLimiter, validate(forgotPasswordSchema), async (req, res, next) => {
  try {
    await passwordResetService.requestReset(req.body.email);
    return res.json({ message: 'If that email is registered, a reset link has been sent.' });
  } catch (err) { next(err); }
});

router.post('/reset-password', resetSubmitLimiter, validate(resetPasswordSchema), async (req, res, next) => {
  try {
    await passwordResetService.resetPassword(req.body.token, req.body.newPassword);
    return res.json({ message: 'Password reset successful. Please log in with your new password.' });
  } catch (err) { next(err); }
});

export { router as authRouter };
```

## 8. Frontend — Reset Password Page (Next.js)

```tsx
// web/app/(auth)/reset-password/page.tsx
'use client';

import { useState, useEffect } from 'react';
import { useRouter, useSearchParams } from 'next/navigation';
import { apiClient } from '@/lib/auth/api-client';

export default function ResetPasswordPage() {
  const router = useRouter();
  const searchParams = useSearchParams();
  const token = searchParams.get('token');

  const [newPassword, setNewPassword] = useState('');
  const [confirmPassword, setConfirmPassword] = useState('');
  const [status, setStatus] = useState<'idle' | 'loading' | 'success' | 'error'>('idle');
  const [error, setError] = useState('');

  useEffect(() => {
    if (!token) {
      setError('Invalid reset link. Please request a new one.');
      setStatus('error');
    }
  }, [token]);

  async function handleSubmit() {
    if (newPassword !== confirmPassword) {
      setError('Passwords do not match');
      return;
    }

    setStatus('loading');
    setError('');

    try {
      await apiClient.post('/auth/reset-password', { token, newPassword }, { skipAuth: true });
      setStatus('success');
      setTimeout(() => router.push('/login'), 3000);
    } catch (err: any) {
      setError(err.message ?? 'Something went wrong');
      setStatus('error');
    }
  }

  if (status === 'success') {
    return (
      <div>
        <h1>Password reset successful</h1>
        <p>Redirecting you to login...</p>
      </div>
    );
  }

  return (
    <div>
      <h1>Set new password</h1>
      <input
        type="password"
        placeholder="New password (min 8 characters)"
        value={newPassword}
        onChange={(e) => setNewPassword(e.target.value)}
      />
      <input
        type="password"
        placeholder="Confirm new password"
        value={confirmPassword}
        onChange={(e) => setConfirmPassword(e.target.value)}
      />
      {error && <p style={{ color: 'red' }}>{error}</p>}
      <button onClick={handleSubmit} disabled={status === 'loading'}>
        {status === 'loading' ? 'Resetting...' : 'Reset Password'}
      </button>
    </div>
  );
}
```

## 9. Response Shapes

### POST /auth/forgot-password — Always 200
```json
200 OK
{ "message": "If that email is registered, a reset link has been sent." }
```
> Even if the email doesn't exist, wrong provider, or deactivated — same response. No information leakage.

### POST /auth/reset-password — Success
```json
200 OK
{ "message": "Password reset successful. Please log in with your new password." }
```

### POST /auth/reset-password — Errors
```json
// Token not found or already used
400 Bad Request
{ "message": "Invalid or expired reset token" }

// Token expired
400 Bad Request
{ "message": "Reset token has expired. Please request a new one." }

// Validation failure
400 Bad Request
{
  "message": "Validation failed",
  "errors": {
    "newPassword": ["Password must be at least 8 characters"]
  }
}
```

## 10. Security Decisions

| Decision | Why |
|---|---|
| `crypto.randomBytes(32)` not `uuid` | UUID v4 has only 122 bits of randomness and follows a known format. `randomBytes(32)` = 256 bits, fully random |
| Token stored hashed | DB breach doesn't expose valid reset tokens |
| 15-minute expiry | Short enough to limit exposure window if link intercepted |
| Invalidate previous tokens on new request | Prevents multiple valid tokens existing simultaneously |
| Mark `isUsed = true` BEFORE password update | Prevents replay attacks if update partially fails |
| Revoke all refresh tokens after reset | Password change = potential compromise; force re-auth everywhere |
| Always 200 on forgot-password | Prevents email enumeration |
| `bcrypt.compare` loop on candidates | Unavoidable when you can't look up by raw token — kept safe with `take: 10` limit |
