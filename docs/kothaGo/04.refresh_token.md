# Part 4 — Token Rotation via Refresh Token
## POST /api/v1/auth/refresh

---

## Why Token Rotation?

Access tokens are short-lived (15 min). When they expire, instead of forcing the user to log in again, the client sends the refresh token to get a new access token.

**Without rotation:** the same refresh token works forever until expiry — if it's stolen, the attacker has 7-30 days of silent access.

**With rotation:** every refresh call:
1. Deletes the old refresh token from DB
2. Issues a brand new one

So a stolen token can only be used **once** before it's gone. The moment the real user refreshes, both tokens become invalid — and theft is detected.

---

## Token Family Theft Detection — How it Works

```
Login → family = "abc123", token A stored in DB
    ↓
/refresh with token A → delete A, store token B (same family "abc123")
    ↓
/refresh with token B → delete B, store token C (same family "abc123")

── THEFT SCENARIO ──────────────────────────────────────────────────────────

Attacker steals token B (before legitimate user refreshes)
Attacker sends /refresh with token B → success → gets token C
Legitimate user sends /refresh with token B → tokenId not found in DB (already deleted)
    ↓
Backend sees: "token ID not in DB" = it was already rotated = THEFT DETECTED
    ↓
Revoke entire family "abc123" — all sessions from this login are dead
Both legitimate user + attacker are logged out
User gets notified (optional) and re-authenticates cleanly
```

---

## Flow

```
Client → POST /auth/refresh
         Web: refresh token in HttpOnly cookie (automatic)
         Mobile: Authorization: Bearer <refresh_token>
    ↓
authenticateRefresh middleware
  → extracts raw token from cookie or header
  → verifies JWT signature
  → attaches { sub, email, tokenId, family, rawToken } to req.user
    ↓
tokensService.rotateRefreshToken()
  → findUnique by tokenId
  → not found? → revokeTokenFamily → 403 (theft detected)
  → isRevoked? → revokeTokenFamily → 403
  → expired? → 401
  → bcrypt.compare(rawToken, storedHash) → invalid? → 403
  → delete old token record
  → generate new access token
  → generate + store new refresh token (same family)
    ↓
Web: new refresh token in HttpOnly cookie → return { accessToken }
Mobile: return { accessToken, refreshToken }
```

## 1. Tokens Service — rotateRefreshToken()

```typescript
// backend/src/tokens/tokens.service.ts
import jwt from 'jsonwebtoken';
import * as bcrypt from 'bcrypt';
import { v4 as uuidv4 } from 'uuid';
import { prisma } from '../lib/prisma';

export interface JwtPayload {
  sub: string;
  email: string;
}

export interface RefreshJwtPayload extends JwtPayload {
  tokenId: string;
  family: string;
}

export const tokensService = {
  generateAccessToken(payload: JwtPayload): string {
    return jwt.sign(payload, process.env.JWT_ACCESS_SECRET!, {
      expiresIn: (process.env.JWT_ACCESS_EXPIRY ?? '15m') as any,
    });
  },

  async generateRefreshToken(
    userId: string,
    email: string,
    family: string,
    meta: { userAgent?: string; ipAddress?: string } = {},
  ): Promise<string> {
    const tokenId = uuidv4();
    const expiryDays = parseInt(process.env.JWT_REFRESH_EXPIRY_DAYS ?? '7');

    const payload: RefreshJwtPayload = { sub: userId, email, tokenId, family };

    const rawToken = jwt.sign(payload, process.env.JWT_REFRESH_SECRET!, {
      expiresIn: `${expiryDays}d` as any,
    });

    const tokenHash = await bcrypt.hash(rawToken, 10);

    const expiresAt = new Date();
    expiresAt.setDate(expiresAt.getDate() + expiryDays);

    await prisma.refreshToken.create({
      data: { id: tokenId, tokenHash, family, userId, expiresAt, ...meta },
    });

    return rawToken;
  },

  // ─── Core rotation logic ─────────────────────────────────────────────────────
  async rotateRefreshToken(
    payload: RefreshJwtPayload & { rawToken: string },
    meta: { userAgent?: string; ipAddress?: string } = {},
  ): Promise<{ accessToken: string; refreshToken: string }> {

    // ── 1. Lookup by tokenId ──────────────────────────────────────────────────
    // tokenId is embedded in the JWT payload and is also the PK in our DB.
    // This is a single O(1) lookup — no scanning needed.
    const storedToken = await prisma.refreshToken.findUnique({
      where: { id: payload.tokenId },
    });

    // ── 2. Theft detection ────────────────────────────────────────────────────
    // Record doesn't exist → it was already deleted by a previous rotation.
    // Someone is replaying an old token that was already used.
    // This is the core of token family theft detection.
    if (!storedToken) {
      // Nuke the ENTIRE family — all active sessions derived from this login.
      // Both the attacker and the legitimate user get logged out.
      // Harsh but correct — we cannot know which one is the attacker.
      await tokensService.revokeTokenFamily(payload.family);

      const err: any = new Error(
        'Suspicious activity detected. All sessions have been revoked for your security.',
      );
      err.status = 403;
      throw err;
    }

    // ── 3. Check if already revoked ───────────────────────────────────────────
    // isRevoked = true happens when we explicitly call revokeTokenFamily().
    // This catches the case where a family was already nuked by a previous theft detection
    // but the attacker is still trying the same token.
    if (storedToken.isRevoked) {
      await tokensService.revokeTokenFamily(payload.family);
      const err: any = new Error('Token has been revoked. Please log in again.');
      err.status = 403;
      throw err;
    }

    // ── 4. Check expiry ───────────────────────────────────────────────────────
    // JWT verification in middleware already checks exp, but we also check our DB
    // expiry as a second line of defence in case JWT_REFRESH_SECRET was rotated
    // or someone crafted a token with a very long expiry.
    if (new Date() > storedToken.expiresAt) {
      const err: any = new Error('Refresh token expired. Please log in again.');
      err.status = 401;
      throw err;
    }

    // ── 5. Verify raw token matches stored hash ───────────────────────────────
    // Even if someone got hold of the tokenId, they'd need the raw token to pass this.
    // bcrypt.compare is timing-safe by design.
    const isValid = await bcrypt.compare(payload.rawToken, storedToken.tokenHash);
    if (!isValid) {
      await tokensService.revokeTokenFamily(payload.family);
      const err: any = new Error('Invalid refresh token.');
      err.status = 403;
      throw err;
    }

    // ── 6. Rotate ─────────────────────────────────────────────────────────────
    // Delete the old token record first — this is the "rotation" step.
    // Now if the old token is replayed, step 2 above will catch it.
    await prisma.refreshToken.delete({ where: { id: payload.tokenId } });

    const newAccessToken = tokensService.generateAccessToken({
      sub: payload.sub,
      email: payload.email,
    });

    // New refresh token inherits the SAME family — the lineage stays traceable.
    const newRefreshToken = await tokensService.generateRefreshToken(
      payload.sub,
      payload.email,
      payload.family, // Same family, not a new one
      meta,
    );

    return { accessToken: newAccessToken, refreshToken: newRefreshToken };
  },

  // ─── Revoke entire token family ──────────────────────────────────────────────
  // Used when theft is detected — marks all tokens in the family as revoked.
  async revokeTokenFamily(family: string): Promise<void> {
    await prisma.refreshToken.updateMany({
      where: { family },
      data: { isRevoked: true },
    });
  },

  // ─── Revoke on logout ────────────────────────────────────────────────────────
  async revokeUserTokens(userId: string, tokenId?: string): Promise<void> {
    if (tokenId) {
      // Single device logout — only delete this session's token
      await prisma.refreshToken.deleteMany({ where: { id: tokenId, userId } });
    } else {
      // All devices logout — revoke every session for this user
      await prisma.refreshToken.updateMany({
        where: { userId },
        data: { isRevoked: true },
      });
    }
  },

  // ─── Cleanup (run as daily cron) ─────────────────────────────────────────────
  async cleanExpiredTokens(): Promise<void> {
    await prisma.refreshToken.deleteMany({
      where: { expiresAt: { lt: new Date() } },
    });
  },
};
```

## 2. Refresh Middleware

```typescript
// backend/src/middleware/authenticate.ts

export function authenticateRefresh(req: AuthRequest, res: Response, next: NextFunction) {
  // Web sends via HttpOnly cookie (automatically attached by browser)
  // Mobile sends via Authorization: Bearer header
  const rawToken =
    req.cookies?.refresh_token ?? req.headers.authorization?.split(' ')[1];

  if (!rawToken) {
    return res.status(401).json({ message: 'No refresh token provided' });
  }

  try {
    // Verify the JWT signature + expiry
    const payload = jwt.verify(rawToken, process.env.JWT_REFRESH_SECRET!) as any;

    // CRITICAL: attach rawToken to req.user
    // The service needs the raw token string for bcrypt.compare — without it,
    // we can't validate that this specific token (not just any valid JWT) matches
    // what we stored in the DB.
    req.user = { ...payload, rawToken };
    next();
  } catch (err: any) {
    const message =
      err.name === 'TokenExpiredError' ? 'Refresh token expired' : 'Invalid refresh token';
    return res.status(401).json({ message });
  }
}
```

## 3. Auth Router — /refresh route

```typescript
// backend/src/auth/auth.router.ts  (refresh portion)
import { Router, Request, Response, NextFunction } from 'express';
import { tokensService } from '../tokens/tokens.service';
import { authenticateRefresh, AuthRequest } from '../middleware/authenticate';

const router = Router();

const IS_PRODUCTION = process.env.NODE_ENV === 'production';

const REFRESH_COOKIE_OPTIONS = {
  httpOnly: true,
  secure: IS_PRODUCTION,
  sameSite: 'strict' as const,
  maxAge: 7 * 24 * 60 * 60 * 1000,
  path: '/api/v1/auth',
};

function isMobile(req: Request): boolean {
  return /ReactNative|Expo|okhttp/.test(req.headers['user-agent'] ?? '');
}

function getMeta(req: Request) {
  return {
    userAgent: req.headers['user-agent'],
    ipAddress: (req.headers['x-forwarded-for'] as string) ?? req.ip,
  };
}

// POST /api/v1/auth/refresh
router.post(
  '/refresh',
  authenticateRefresh, // Validates JWT signature, attaches payload + rawToken to req.user
  async (req: AuthRequest, res: Response, next: NextFunction) => {
    try {
      const { accessToken, refreshToken } = await tokensService.rotateRefreshToken(
        req.user as any,
        getMeta(req),
      );

      if (!isMobile(req)) {
        // Overwrite the old refresh token cookie with the new one
        res.cookie('refresh_token', refreshToken, REFRESH_COOKIE_OPTIONS);
        // Web only gets access token in body — refresh token is in the cookie
        return res.json({ accessToken });
      }

      // Mobile gets both — they'll save refreshToken to Keychain/EncryptedStorage
      return res.json({ accessToken, refreshToken });
    } catch (err) {
      next(err);
    }
  },
);

export { router as authRouter };
```

## 4. Client-Side Token Refresh

### Next.js — Auto-Refresh on 401

```typescript
// web/lib/auth/token-manager.ts
let accessToken: string | null = null;
let refreshPromise: Promise<string | null> | null = null;

export const tokenManager = {
  setAccessToken(token: string) { accessToken = token; },
  getAccessToken() { return accessToken; },
  clearAccessToken() { accessToken = null; },

  // Promise deduplication — if 3 concurrent requests all get 401,
  // only ONE refresh call is made. All 3 await the same promise.
  // Without this, you'd send 3 refresh requests simultaneously,
  // and the 2nd + 3rd would fail (their tokens are already rotated by the 1st).
  async refreshAccessToken(): Promise<string | null> {
    if (refreshPromise) return refreshPromise;

    refreshPromise = fetch('/api/auth/refresh', {
      method: 'POST',
      credentials: 'include', // Sends the HttpOnly cookie automatically
    })
      .then(async (res) => {
        if (!res.ok) {
          accessToken = null;
          return null;
        }
        const data = await res.json();
        accessToken = data.accessToken;
        return data.accessToken;
      })
      .finally(() => {
        refreshPromise = null; // Reset so next expiry triggers a fresh call
      });

    return refreshPromise;
  },
};
```

```typescript
// web/lib/auth/api-client.ts
import { tokenManager } from './token-manager';

const API_BASE = process.env.NEXT_PUBLIC_API_URL ?? 'http://localhost:3001/api/v1';

async function fetchWithAuth(endpoint: string, options: RequestInit & { skipAuth?: boolean } = {}) {
  const { skipAuth = false, ...rest } = options;

  const makeRequest = (token?: string | null) =>
    fetch(`${API_BASE}${endpoint}`, {
      ...rest,
      credentials: 'include',
      headers: {
        'Content-Type': 'application/json',
        ...(token && !skipAuth ? { Authorization: `Bearer ${token}` } : {}),
        ...rest.headers,
      },
    });

  let res = await makeRequest(tokenManager.getAccessToken());

  // 401 = access token expired → attempt silent refresh once
  if (res.status === 401 && !skipAuth) {
    const newToken = await tokenManager.refreshAccessToken();
    if (!newToken) {
      // Refresh also failed (expired or revoked) → force re-login
      window.location.href = '/login';
      return res;
    }
    res = await makeRequest(newToken);
  }

  return res;
}

export const apiClient = {
  async get<T>(endpoint: string): Promise<T> {
    const res = await fetchWithAuth(endpoint);
    if (!res.ok) throw new Error((await res.json()).message);
    return res.json();
  },
  async post<T>(endpoint: string, body: unknown, opts?: RequestInit): Promise<T> {
    const res = await fetchWithAuth(endpoint, {
      method: 'POST',
      body: JSON.stringify(body),
      ...opts,
    });
    if (!res.ok) throw new Error((await res.json()).message);
    return res.json();
  },
};
```

### React Native — Auto-Refresh on 401

```typescript
// mobile/src/auth/api-client.ts
import { tokenStorage } from './token-storage';

const API_BASE = process.env.EXPO_PUBLIC_API_URL ?? 'http://localhost:3001/api/v1';

let accessToken: string | null = null;
let refreshPromise: Promise<string | null> | null = null;

export const mobileTokenManager = {
  setAccessToken(t: string) { accessToken = t; },
  getAccessToken() { return accessToken; },
  clearAccessToken() { accessToken = null; },

  async refreshAccessToken(): Promise<string | null> {
    if (refreshPromise) return refreshPromise;

    refreshPromise = (async () => {
      const refreshToken = await tokenStorage.getRefreshToken();
      if (!refreshToken) return null;

      const res = await fetch(`${API_BASE}/auth/refresh`, {
        method: 'POST',
        headers: {
          Authorization: `Bearer ${refreshToken}`,
          'User-Agent': 'ReactNative/LangLearn',
        },
      });

      if (!res.ok) {
        await tokenStorage.deleteRefreshToken();
        return null;
      }

      const data = await res.json();
      accessToken = data.accessToken;

      // Save the new rotated refresh token to Keychain
      if (data.refreshToken) {
        await tokenStorage.saveRefreshToken(data.refreshToken);
      }

      return data.accessToken;
    })().finally(() => { refreshPromise = null; });

    return refreshPromise;
  },
};

export async function mobileFetch(endpoint: string, options: RequestInit = {}) {
  const makeRequest = (token?: string | null) =>
    fetch(`${API_BASE}${endpoint}`, {
      ...options,
      headers: {
        'Content-Type': 'application/json',
        ...(token ? { Authorization: `Bearer ${token}` } : {}),
        ...options.headers,
        'User-Agent': 'ReactNative/LangLearn',
      },
    });

  let res = await makeRequest(mobileTokenManager.getAccessToken());

  if (res.status === 401) {
    const newToken = await mobileTokenManager.refreshAccessToken();
    if (!newToken) throw new Error('SESSION_EXPIRED');
    res = await makeRequest(newToken);
  }

  return res;
}
```

```json
200 OK
```

```json
// Token already rotated (theft detected)
403 Forbidden
{ "message": "Suspicious activity detected. All sessions have been revoked for your security." }

401 Unauthorized

// No token sent at all
401 Unauthorized
{ "message": "No refresh token provided" }
```

## 6. Session Limit (Optional Hardening)

Prevent a single user from accumulating unlimited concurrent sessions — useful for shared/compromised devices.

```typescript
// In tokensService.generateRefreshToken — add this before prisma.refreshToken.create()

const MAX_SESSIONS = 5;

const existingCount = await prisma.refreshToken.count({
  where: { userId, isRevoked: false },
});

if (existingCount >= MAX_SESSIONS) {
  // Revoke the oldest active session to make room
  const oldest = await prisma.refreshToken.findFirst({
    where: { userId, isRevoked: false },
    orderBy: { createdAt: 'asc' },
  });

  if (oldest) {
    await prisma.refreshToken.delete({ where: { id: oldest.id } });
  }
}
```
